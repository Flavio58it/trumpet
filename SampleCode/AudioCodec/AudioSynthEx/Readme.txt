---------------------------------------------------------------------------------------------------------
Purpose:
---------------------------------------------------------------------------------------------------------
		1. Demo how to playback an audio file from SPI flash
		2. Demo how to playback an equation which represents a group of audio files from SPI flash

---------------------------------------------------------------------------------------------------------
Operation:
---------------------------------------------------------------------------------------------------------
	LED:
		1. 	GPA14: show system standby
		2. 	GPA15: show equation mode
		3. 	GPB4: use firmware PWM to show UltraIo first  curve 
		4. 	GPB5: use hardware PWM to show UltraIo second curve

	Key:
		1. 	GPB0: start or stop playback
		2. 	GPB1: play the next audio file or the next equation
		3. 	GPB2: play the previous audio file or the previous equation
		4. 	GPB3: power down(keep pressing than 4 second)/wake up
		5.	GPB6: select playing an audio file or an equation
		6.	GPB7: Pause or resume		

---------------------------------------------------------------------------------------------------------
Note:
---------------------------------------------------------------------------------------------------------
	Note I:
		1. 	Before playing, there must be written the "AudioRes_AudioInfoMerge.ROM" file into SPI flash.
		   	The "AudioRes_AudioInfoMerge.ROM" is storted in ".\AudioRes\Output\"
		2. 	The encoded data can be generated by AudioTool.exe under Windows\tool\audioTool\.
		   	There is an audio tool project "AudioRes.wba" located in ".\AudioRes\".
		   	the "AudioRes_AudioInfoMerge.ROM" file is created by this project
		3. 	The playback sampling rate is read from audio file or equation
		4. 	Playback can be with/without N-times up-sampling to reduce "metal sound"
		   	which is controlled by APU_UPSAMPLE defined in "ConfigApp.h"	
 		5.	If you want to create new curve, please use "UltraIO tool" to create "ConfigUltraIO.h" 
			and replace local file with new "ConfigUltraIO.h".

	Note II:
		1. 	User can enable options to invoke codec for ROM file content, as follow:
			#define P16_ENABLE	 		(1)	 ; 1:enable  0:disable	
			#define LP8_ENABLE	 		(1)	
			#define MD4_ENABLE	 		(1)	
			#define MIDI_ENABLE	 		(1)	
			#define NUONE_ENABLE	 		(0)	
			#define NUSOUND_ENABLE	 		(0)	
			#define NULITE_ENABLE	 		(0)
			#define NUVOX53_ENABLE	 		(0)	
			#define NUVOX63_ENABLE	 		(0)
			#define IMAADPCM_ENABLE	 		(1)
		2.	User needs to estimate code size and ram size if all codecs are invoked!
		3. 	The read callback function,ReadDataCallback(), is implemented in "AppCallback.c" 
		4. 	The event callback function,UserEventCallback(), is implemented in "AppCallback.c"

---------------------------------------------------------------------------------------------------------
Must Know:
---------------------------------------------------------------------------------------------------------
	1.	The stack size is defined by "Stack_Size" in "startup_ISD9100.s". Can change the stack size
		by modifing the content of "Stack_Size".
	2.	The chip ROM size and SRAM size is specified by "ROM_SIZE" and "SRAM_SIZE"in  linker settings
		Open the linker setting and find the --pd="-DSRAM_SIZE=0xNNNN" --pd="-DROM_SIZE=0xMMMMM" in "misc controls" 
	3.	When the error happen:
			"L6388E: ScatterAssert expression (ImageLimit(_SRAM) < (0xNNNNNNNN + 0xMMMM)) failed on line xx" 
		It means the size of variables+stack is over chip SRAM size.
		Please open map file to see the overed SRAM size and reduce variables or stack size to fit chip SRAM size.
	4.	When the error happen:
			"L6220E: Load region _ROM size (NNNNN bytes) exceeds limit (MMM bytes)."
		It means the size of programs is over chip ROM size.
		Please open map file to see the overed programs size and reduce programs size to fit chip ROM size.
